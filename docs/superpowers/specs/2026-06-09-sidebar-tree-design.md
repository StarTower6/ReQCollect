# SideBar 树状工作空间与会话管理 — 设计文档

## 1. 概述

将当前左侧栏（SideBar）从扁平会话列表改造为树状结构：工作空间为可展开/折叠的父节点，会话为叶子节点。目标是让用户按项目组织视角管理会话，而不是在全局扁平列表中翻找。

## 2. 交互规则

### 2.1 树状展开 — 互斥模式

- **任何时候只有一个工作空间可以展开**（互斥展开，手风琴模式）
- 点击折叠状态的工作空间 → 展开该工作空间，同时自动折叠之前展开的其他工作空间
- 点击已展开的工作空间标题 → 折叠它（全部收起，无展开项）
- **「未分类会话」特殊处理**：固定在最后，不与工作空间争抢焦点，默认展开，但也可手动折叠
- 展开/折叠使用 `▶` 箭头指示器，展开时旋转 90° 为 `▶`

### 2.2 工作空间节点

每个工作空间节点显示：
- `📁` 图标 + 工作空间名称
- 蓝色气泡数字：该工作空间下的会话数量
- 点击名称区域触发展开/折叠
- 点击名称区域以外的操作（如右键菜单）暂留后续

### 2.3 会话叶子节点

每个会话叶子节点显示：
- `💬` 图标 + 会话名称（project_name）
- 右侧显示相对时间（"2天前"、"刚刚"）
- 活跃会话高亮（蓝色背景）
- 点击即切换会话：跳转到 `/chat/{sessionId}`

### 2.4 新对话

- SideBar 顶部保留「新对话」按钮（设计为 `＋` 按钮）
- 如果当前有展开的工作空间，新会话自动关联到该 workspace
- 如果没有任何工作空间展开，新会话进入「未分类会话」

### 2.5 搜索

- 搜索输入框保留在顶部
- 搜索时**所有工作空间自动展开**，显示匹配的会话
- 清除搜索后恢复到原来的展开状态
- 搜索匹配会话名称和会话 ID

### 2.6 新建工作空间

- SideBar 底部新增「＋ 新建工作空间」入口（虚线边框按钮）
- 点击跳转到工作空间列表页，或弹出快速创建 dialog

## 5. 组件改动

### 5.1 SideBar.vue — 重写

| 部分 | 说明 |
|------|------|
| 顶部区域 | Brand + 新对话按钮 + 搜索框 |
| 新建工作空间 | 虚线按钮（跳转到 WorkspaceList 或内联创建） |

### 5.2 AppLayout.vue — 微调

- 挂载时同时加载 workspaces（新增 API 调用）
- handleNewChat 传递当前展开的 workspaceId

### 5.3 其他组件

- `WorkspaceDetail.vue` 中的 "新建会话" 不再需要独立的 `setWorkspace` + `newSession` 逻辑，统一走 SideBar

## 6. 设计边界说明

### 6.1 不做

- **不实现拖拽排序** — 工作空间排序暂按 `updated_at`
- **不实现工作空间内会话的拖拽移动** — M1 发布后再考虑
- **不实现右键菜单** — 工作空间/会话的操作入口在详情页完成
- **不实现 inline 重命名** — 通过详情页编辑
- **不实现无限的树嵌套** — 仅两层（工作空间 → 会话）

### 6.2 异常状态处理

- **工作空间列表为空**：仅显示「未分类会话」，不显示虚线按钮新建按钮（因为新建后要刷新页面）
- **会话列表为空**：工作空间节点显示 `0`，展开后显示空状态提示
- **工作空间加载失败**：静默降级，仅展示已缓存的 session 列表（扁平视图兜底）
- **搜索无结果**：显示「未找到匹配的会话」
