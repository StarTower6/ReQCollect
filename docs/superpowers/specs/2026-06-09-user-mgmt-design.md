# 用户管理 UI 改造 — 方案 C 设计文档

## 1. 概述

基于现有用户管理页面进行 UI 改造，采用方案 C 风格：紧凑统计条 + 极简表格 + 侧边抽屉详情。

## 2. 布局结构

```
┌──────────────────────────────────────────────────────┐
│ ← 返回   用户管理                        [+ 新增]     │
├──────────────────────────────────────────────────────┤
│ 👥 总 12  │  ✅ 活跃 10  │  🔒 管理员 2  │  🏢 部门 6  │
├──────────────────────────────────────────────────────┤
│ [□] 用户            部门    角色    状态    操作    │
├──────────────────────────────────────────────────────┤
│ [□] 👤 管理员        IT部   管理员  ● 活跃   详情  │
│ [□] 👤 张伟          财务部  业务方  ● 活跃   详情  │
│ [□] 👤 李丽          质量部  审核人  ○ 禁用   详情  │
│ ...                                                  │
├──────────────────────────────────────────────────────┤
│ 已选 0 项                        ← 1 2 3 ... →       │
└──────────────────────────────────────────────────────┘
```

## 3. 交互规则

### 3.1 统计条

- 单行紧凑设计：`标签 数字` 格式
- 4 个指标：总用户、活跃、管理员、部门
- 指标数值用粗体强调

### 3.2 表格

- 5 列：复选框 / 用户 / 部门 / 角色 / 状态 / 操作
- 每行可点击「详情」或在 Element Plus 表格中单击行触发
- 禁用用户（`is_active === false`）整行灰色显示
- 角色用彩色 tag 区分
- 状态列用绿色/灰色圆点 + 文字

### 3.3 批量操作

- 表头复选框可全选
- 底部左侧显示「已选 N 项」
- 选中后操作栏显示「批量禁用 / 批量启用 / 批量删除」

### 3.4 侧边抽屉详情

- 点击「详情」或双击行打开 `el-drawer`（右侧，宽度 420px）
- 抽屉内容：
  - 顶部：用户名 + 角色 tag
  - 字段：显示名、邮箱、部门、来源、创建时间
  - 状态切换开关
  - 底部：保存 + 删除按钮
- 保存和删除调用现有 API

### 3.5 筛选

- 搜索框：搜索用户名、显示名、邮箱
- 角色下拉筛选
- 状态下拉筛选

## 4. API 依赖

现有后端 API 完整，无需新增后端代码：

| 操作 | 端点 | 方法 |
|------|------|------|
| 列表 | `/api/auth/users` | GET |
| 新增 | `/api/auth/register` | POST |
| 编辑 | `/api/auth/users/{id}` | PATCH |
| 状态切换 | `/api/auth/users/{id}/status` | PATCH |
| 删除 | `/api/auth/users/{id}` | DELETE |

## 5. 改动范围

- **修改**: `reqcollect-web/src/views/admin/UsersView.vue` — 全量重写 template + style
- **无后端改动**: API 层完整
