# 01 — 对话挖需引擎

## 概述

多轮对话引导业务人员表达需求，AI 逐步填充需求画像，达到完整度阈值后触发 PRD 生成。

## 当前状态 ✅ 已实现

### ReAct Agent
- [x] LangGraph ReAct 循环，支持工具调用
- [x] 6 个 Agent 工具（更新画像、查看摘要、评估完整度、待追问、时间、知识检索 stub）
- [x] MemorySaver 内存检查点（无需 Redis）
- [x] SSE 流式输出，逐 token 渲染

### 人设与引导
- [x] "资深企业 IT 需求分析师"人设提示词
- [x] 每次 1-2 个问题，带【单选】/【多选】选项引导
- [x] 先总结已知信息再提问

### 完整度评估
- [x] 11 个业务字段加权评分
- [x] 字段：project_name, business_background, current_process, user_roles, business_flow, functional_requirements, existing_systems, non_functional, data_scale, constraints, success_criteria
- [x] 阈值 0.75 → 建议生成 PRD

### 规则提取器（保底）
- [x] 正则提取项目名称、角色、系统、功能关键词

## 待实现 🔲

### 对话体验
- [ ] **追问策略控制** — 预定义分阶段追问顺序（先背景→再角色→再流程→再技术），避免 LLM 随机跳转
- [ ] **上下文记忆优化** — 长对话压缩，避免超过 LLM 上下文窗口
- [ ] **用户打断处理** — 用户可随时换话题，Agent 能平滑切换
- [ ] **不确定选项** — 每个问题必须有"不确定/您觉得呢"兜底选项

### 知识库
- [ ] **知识检索启用** — 对接向量库（Milvus/FAISS），检索历史项目经验
- [ ] **相似项目推荐** — 基于用户描述推荐历史相似项目画像

### 画像增强
- [ ] **画像冲突检测** — 用户前后说法不一致时提示确认
- [ ] **画像导出** — 需求画像可导出为结构化 JSON/Excel

### 评估优化
- [ ] **LLM 评估模式** — 对于复杂需求，启用 LLM 二次评估而非简单公式加权
- [ ] **动态阈值** — 根据项目复杂度动态调整完整度阈值

### 多轮状态
- [ ] **会话恢复** — 用户在企微/Web 中断对话后，可回到上次位置继续
- [ ] **会话标签与搜索** — 给会话打标签（项目类型/部门），支持模糊搜索
