# 03 — 用户与权限

## 概述

当前系统无用户概念，所有人共用。企业级必须支持用户隔离、权限分级和审计追踪。

## 当前状态 ✅ 已实现

- [x] 无（从零开始）

## 待实现 🔲

### 用户认证（P0）
- [ ] **登录页面** — 独立登录页面，退出后重新登录
- [ ] **LDAP/OIDC 集成** — 对接公司 AD/LDAP 或 OIDC 统一认证
- [ ] **企业微信扫码登录** — 使用现有 WeCom 身份体系扫码登录
- [ ] **JWT Token** — 登录后签发 Token，API 请求携带认证

### 用户隔离（P0）
- [ ] **个人会话空间** — 用户只能看到自己的会话和 PRD
- [ ] **会话归属** — 每一条消息、画像、PRD 关联用户 ID
- [ ] **用户管理** — 管理员可查看/禁用用户

### 角色权限（P1）
- [ ] **角色模型**
  - **业务方** — 参与对话、查看 PRD、评论
  - **需求分析师** — 管理对话、生成 PRD、发布
  - **审核人** — 审批 PRD、查看所有项目
  - **管理员** — 用户管理、系统设置
- [ ] **权限矩阵** — 每个端点校验角色权限

### 审计日志（P1）
- [ ] **操作日志** — 记录谁在什么时间做了什么（对话、生成、修改、删除）
- [ ] **需求溯源** — 每条需求字段可以追溯到是在哪轮对话中产生的
- [ ] **审计导出** — 按时间范围导出审计日志

### 协作（P2）
- [ ] **会话共享** — 分析师可将会话分享给其他分析师协作
- [ ] **评论/@提及** — 在 PRD 上@某人进行评审
- [ ] **通知** — PRD 生成完成/有新评论时通知相关人（企微消息）
