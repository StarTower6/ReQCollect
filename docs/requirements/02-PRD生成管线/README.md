# 02 — PRD 生成管线

## 概述

基于需求画像，自动生成结构化需求文档（PRD），支持全量生成和增量生成两种模式。

## 当前状态 ✅ 已实现

### 生成管线
- [x] **Planner** — 基于需求画像规划 9 章节
- [x] **Generator** — LLM 逐章流式生成
- [x] **Assembler** — Markdown 全文拼装

### PRD 模板（9 章，企业 IT 方向）
- [x] 1. 项目背景与目标
- [x] 2. 用户角色与用例（含权限矩阵）
- [x] 3. 业务流程（主流程、分支、异常）
- [x] 4. 功能需求（P0/P1 优先级 + 验收条件）
- [x] 5. 系统集成（对接系统、接口、数据流向）
- [x] 6. 非功能需求（性能、安全、合规）
- [x] 7. 数据需求（量级、存储、报表）
- [x] 8. 实施约束与风险
- [x] 9. 验收标准

### 生成模式
- [x] **one_shot** — 全量生成
- [x] **incremental** — 逐章生成，每章需确认后继续（awaiting_approval 事件）

### 文档质量

#### 内容规范（P0）

- [ ] **去技术化描述** — PRD 各章节聚焦业务功能需求，不出现表结构、接口规范、SQL、配置参数等技术术语
- [ ] **业务视角** — 数据需求章节描述业务数据（"每天产生多少条记录"、"按什么维度统计"），不写数据字典；系统集成章节描述"什么系统之间传什么数据"，不写 API 协议
- [ ] **业务方可验收** — 验收标准从业务视角编写，业务方可以直接判断"做完了还是没做"
- [ ] **一致性检查** — PRD 各章节间交叉引用一致性校验（术语、角色名称）
- [ ] **完整度检查** — 画像中的字段是否全部覆盖到 PRD 各章节
- [ ] **格式检查** — Markdown 格式规范、标题层级、列表对齐

### 版本管理
- [ ] **版本历史** — 每次生成的 PRD 自动归档，支持版本对比（diff）
- [ ] **修订模式** — 用户可针对某章节修改，生成 v2 版本而非全量重来
- [ ] **变更记录** — 自动生成 Change Log：v1→v2 改了什么

### 模板定制
- [ ] **自定义模板** — 用户可上传自己的 PRD 模板（Markdown 模板文件）
- [ ] **章节开关** — 可启用/禁用特定章节
- [ ] **多模板** — 按项目类型（新建系统/系统改造/数据项目）使用不同模板

### 评审流转
- [ ] **评审链接** — 生成可分享的评审链接（业务方打开即可评论）
- [ ] **批注系统** — 业务方/开发方在 PRD 上添加批注
