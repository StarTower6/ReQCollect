# PGE 三段式工作法 — 总纲

所有任务严格遵循 **Plan → Generate → Evaluate** 三段式，不允许跳过任何阶段。

## 核心流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  计划    │ ──→ │  生成    │ ──→ │  评估    │
│  Plan    │     │ Generate │     │ Evaluate │
└──────────┘     └──────────┘     └──────────┘
     ↑                               │
     └──────────── 循环 ──────────────┘
```

## 强制执行规则

1. **Plan 阶段必须先输出** — 展示计划后再执行，除非用户说"直接做"
2. **每个阶段完成需确认** — 完成一个阶段后提示用户确认再进入下一阶段
3. **Evaluate 不可跳过** — 生成完成后必须评估，不能"我觉得没问题"
4. **失败回退** — 评估发现问题时返回 Plan 阶段重新规划

## 设计哲学（借鉴 Hermes Agent 的 Skill 模式）

- **知识优先** — 执行前先加载相关技能（design, impeccable, gitnexus）
- **渐进式披露** — 先给概览再深入细节，不一股脑丢信息
- **工具优先** — 能用脚本解决的不用手动操作
- **记忆复用** — 记录项目约定和常见陷阱，避免重复犯错
