# PGE 三段式 — Plan（计划阶段）

## 目标

在动手写代码之前充分理解任务范围、依赖关系和风险点。

## 必需输出

输出一个实施计划，包含以下内容：

```markdown
## 实施计划

### 1. 任务理解
- 需求来源: docs/requirements/<模块>/README.md
- 核心目标: 一句话概括

### 2. 改动清单
- 新增文件: [...]
- 修改文件: [...]
- 删除文件: [...]

### 3. 数据模型 (如有)
- 新增表/字段

### 4. 风险与依赖
- 依赖关系
- 注意事项

### 5. 实施步骤 (按顺序)
1. ...
2. ...
```

## 原则

- **读需求** — 必须先读 `docs/requirements/` 下的对应文档
- **尊重现有架构** — 不要推翻现有结构重写
- **小步提交** — 一个功能一次 commit，不要攒一堆一次性提交
- **先确认后动手** — 把计划展示给用户，等确认后再进入 Generate
- **技能准备** — 如果涉及前端设计，先加载 design / impeccable 技能
