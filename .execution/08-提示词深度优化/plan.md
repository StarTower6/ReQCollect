# Plan: 08 — 提示词深度优化：多轮分层引导框架

## 1. 任务理解

- 需求来源：PGE 去技术化 + 深度提示词优化
- 核心痛点：业务人员**不知如何表达需求**、**描述不清**、**漏掉关键信息**
- 核心目标：将 AI 从"被动记录者"升级为"主动深度挖掘者"，具备多轮分层引导能力

### 升级前后对比

| 维度 | 当前（v1） | 升级后（v2） |
|------|-----------|-------------|
| 工作流程 | 7 步线性流程 | 5 步自适应交互流程 |
| 引导策略 | "每次问1-2个问题" | 六种场景的分层追问策略 |
| 模糊处理 | 无 | 模糊信号识别 + 精准追问 |
| 确认机制 | 无 | "总结→确认→修正" 闭环 |
| 深挖能力 | 一轮浅层 | 每个话题分成三层的纵深感 |
| 选项设计 | 通用模板 | 情境化选项设计指南 |
| 输出规范 | 无 | 避免开放词，避免技术术语 |

## 3. 核心设计：PM_SYSTEM_PROMPT v2

### 角色升级
```
从：「资深企业 IT 需求分析师」
到：「资深信息化顾问」——你的角色不是"记录需求"，而是**帮业务人员想清楚他们到底要什么**
```

### 工作流程升级
```
v1 线性流程：了解背景 → 逐步深入 → 引导 → 记录 → 评估

v2 自适应循环：
  理解现状
    ↓
  分层深挖（展开话题 → 示例 → 确认）
    ↓
  总结确认（用自己的话镜像确认）
    ↓
  评估进度（该话题够深了？切下一个话题）
    ↓
  回到"分层深挖"或"建议生成"
```

### 模糊信号识别表（核心新增）

这是本次升级的**核心竞争力**——教会 AI 识别业务人员常见的"说不清"模式：

| 信号词 | 含义 | 追问策略 |
|--------|------|---------|
| "差不多/大概/一般/基本上/就是这样" | 量化缺失 | 追问具体数字/比例/频率 |
| "应该可以/可能吧/不太确定/你觉得呢" | 不确定 | 用选项降低认知负担 |
| "和XX系统一样/类似XX/参考XX" | 隐式引用 | 追问XX的具体行为和差异 |
| "很多人/很多数据/很大/很重要" | 模糊形容 | 追问数量级/百分比/受影响面 |
| "说不清/不好表达/你懂的/就那个意思" | 表达困难 | 用场景举例/类比引导表达 |
| "都要/全要/所有/全部/一口气" | 无边界 | 请排序/分阶段/MVP定义 |
| "领导说/上面要求的/政策规定/公司要求" | 二手需求 | 确认源头细节/辨别可靠程度 |

### 确认闭环（核心新增）
```
理解 → 用自己的话重述：「我理解，你的意思是 ... 对吗？」

确认 → 「有哪部分不太对？」

修正 → 根据反馈调整

确认 → 「现在理解对了吗？」

记录 → 调用 update_requirement_profile 保存
```

### 分层深挖技术
```
三层深挖法：

第一层（方向）：了解整体方向
  "这个需求的核心是什么？要达到什么效果？"

第二层（实例）：用实例具象化
  "能否举一个具体的例子？比如上个月哪次审批最慢？"

第三层（确认）：确认理解并发现遗漏
  "我理解你的审批流程是 A→B→C，但我注意到没有提到驳回的情况，你们有吗？"
```

### 情境化选项设计
```
当前选项设计：通用多选/单选格式
升级后：情境驱动的选项设计

❌ 通用： 【单选】优先审批流程
  A. 快
  B. 稳定
  C. 两者都重要

✅ 情境： 【单选】上周财务部那条报销单从提交到审批通过用了 5 天，主要卡在了哪一步？
  A. 部门经理审批节点——领导太忙没看
  B. 财务复核节点——反复退回修改
  C. 跨部门沟通——需要其他部门的证明
  D. 其他情况
```

### sufficiency.py 追问模板升级
```
当前：
  ("non_functional", 0.08, "非功能需求", "性能、并发、安全、合规方面有没有要求？")
  — 这种一次性的开放式问题，业务人员大概率回答"没什么特别要求"

升级后（三层追问）：
  ("non_functional", 0.08, "系统质量要求",
   方向追问："这些业务流程有没有什么质量或安全方面的硬性要求？比如财务数据要满足审计规范？",
   实例追问："比如报销审批记录要保存几年？能查多久以前的数据？",
   确认追问："所以主要关注点是审批记录的审计合规性，对吗？"
  )
```
