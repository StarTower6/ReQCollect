# ReQCollect — 执行流程

## 目录结构

```
.execution/
├── README.md                    ← 本文件
├── <feature-name>/              ← 每次功能开发的执行流程
│   ├── plan.md                  ← 计划阶段（可多轮修订）
│   ├── tasks.md                 ← 生成阶段任务清单
│   └── report.md                ← 评估阶段报告
└── archive/                     ← 已完成的流程归档
    └── <feature-name>/          ← 同上结构
```

## 完整执行流程

```
用户提出需求
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  Plan 阶段                                            │
│  ┌─────────────────────────────────────────────────┐ │
│  │  1. 读 docs/requirements/<模块>/README.md        │ │
│  │  2. 写 plan.md（改动清单 + 验收标准 + 实施步骤） │ │
│  │  3. 给用户看 → 用户提修改意见 → 改 plan.md      │ │
│  │  4. 重复 2-3 直到用户确认"进入下一步"            │ │
│  └─────────────────────────────────────────────────┘ │
│                                                       │
│  用户确认后                                           │
│    ▼                                                  │
├─────────────────────────────────────────────────────┤
│  Generate 阶段                                        │
│  ┌─────────────────────────────────────────────────┐ │
│  │  1. 从 plan.md 提取任务清单 → tasks.md           │ │
│  │  2. 任务可分配给多个子 agent 并行执行            │ │
│  │  3. 严格按任务清单执行，不做计划外的              │ │
│  └─────────────────────────────────────────────────┘ │
│                                                       │
│  生成完成后                                           │
│    ▼                                                  │
├─────────────────────────────────────────────────────┤
│  Evaluate 阶段                                        │
│  ┌─────────────────────────────────────────────────┐ │
│  │  1. 对照 tasks.md 逐条验证完成情况               │ │
│  │  2. 对照 plan.md 中的验收标准逐条打勾             │ │
│  │  3. 输出 report.md                               │ │
│  │  4. P0 全过 → 通过                               │ │
│  │  5. 不通过 → 回退 Plan 阶段                      │ │
│  └─────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

## 约束

- 每个功能开发一个文件夹，不要混在一起
- plan.md 允许多轮修改，每轮保留修订历史（用 ## v2、## v3 标注）
- tasks.md 允许拆分多个子任务并行执行
- report.md 必须逐条对照验收标准打勾
- 归档: 完成后将文件夹移到 archive/，加日期前缀
